## Documentação da API FlowApplications v1

Bem-vindo à documentação da API FlowApplications! Nossa API permite que você integre facilmente pagamentos PIX dinâmicos em seu sistema usando a Woovi.

## Base URL

Todas as requisições devem ser feitas para a seguinte base URL:

```
https://www.flowapplications.com.br/api/v1
```

## Autenticação

Todos os endpoints (exceto o endpoint de QR code) requerem autenticação via Bearer Token no header:

```
Authorization: Bearer {seu_token_api}
```

O token da API pode ser gerado através do dashboard após aprovação da solicitação de acesso à API.

<Warning>
Mantenha seu token de API seguro e nunca o compartilhe publicamente. O token deve ser mantido como uma variável de ambiente ou configuração segura.
</Warning>

## Status de Pagamento

Os pagamentos podem ter os seguintes status:

- **ACTIVE**: Pagamento pendente, aguardando pagamento
- **COMPLETED**: Pagamento confirmado e concluído
- **EXPIRED**: Pagamento expirado, não pode mais ser pago

## Códigos de Status HTTP

### 2xx - Sucesso

- **200 OK**: Requisição bem-sucedida

### 4xx - Erro do Cliente

- **400 Bad Request**: Parâmetros inválidos ou faltando
- **401 Unauthorized**: Token de autenticação inválido, expirado ou não fornecido
- **404 Not Found**: Recurso não encontrado (pagamento, QR code, etc)

### 5xx - Erro do Servidor

- **500 Internal Server Error**: Erro interno do servidor, configuração faltando ou erro na comunicação com a Woovi

## Limites e Restrições

- **Valor mínimo**: R$ 0,01
- **Valor máximo**: Sem limite
- **Tempo de expiração mínimo**: 300 segundos (5 minutos)
- **CorrelationID**: Deve ser único para cada pagamento. Use um formato que garanta unicidade (ex: UUID, timestamp, ID do pedido)
- **Rate Limits**: Respeite os limites de requisições. Em caso de abuso, o acesso à API pode ser suspenso

## Notas Importantes

1. **QR Code Image**: O campo `qrCodeImage` retorna uma URL para acessar a imagem PNG do QR code diretamente. Você pode usar essa URL em tags `<img>` ou baixar a imagem.

2. **paymentLinkUrl**: Este campo foi **removido** da resposta da API. Use o `qrCodeImage` ou o `brCode` para realizar o pagamento.

3. **CorrelationID**: Deve ser único para cada pagamento. Use um formato que garanta unicidade (ex: UUID, timestamp, ID do pedido).

4. **Valores**: Os valores são sempre em reais (R$). A API converte automaticamente para centavos ao enviar para a Woovi.

5. **Expiração**: Se não fornecer `expiresIn` ou `expiresDate`, a Woovi pode usar um tempo padrão. Recomendamos sempre especificar o tempo de expiração.

6. **Webhooks**: Consulte a documentação sobre webhooks para receber notificações automáticas quando um pagamento for confirmado.
